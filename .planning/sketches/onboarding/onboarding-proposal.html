Принципы

Что делаем

  • Показываем, не рассказываем — слайд 1 это живой мини-диалог с ботом, не абстрактная картинка.
  • Доверие до доступа — «вы контролируете каждый ответ» звучит до того, как просим подключить WhatsApp или Telegram.
  • Оба канала равноправны — блок доверия и экран успеха подстраиваются под канал, выбранный в визарде.
  • Прайс — после авторизации — экран успеха сразу предлагает загрузить прайс-лист; первый ответ бота закрывает чек-лист.
  • Чек-лист доводит до ценности — прайс-лист и первый ответ — то, что превращает установку в работающего бота.

Чего не делаем

  • Никакой регистрации/логина — приложение работает без аккаунта, это преимущество, не трогаем.
  • Без кнопки «Пропустить» на слайдах — три информационных экрана проходятся кнопкой «Далее»; выход из потока один — «Создать бота».
  • Не блокируем повторный вход — карусель только при первом запуске (PlayerPrefs-флаг), повторно доступна из «О приложении».
  • Не дублируем визард — AddBotPanel уже хороший, онбординг обрамляет его, а не заменяет.
  • Без видео и лотти — статичные композиции + DOTween достаточно, вес приложения не растёт.

Реализация (кратко)

Структура

  • Screen_Onboarding — новый экран, строится билдером OnboardingScreenBuilder по паттерну NavRestructureBuilder; в ScreenContainer до auth-экранов (WhatsappAuth/TelegramAuth остаются последними).
  • Карусель — существующий SnappyFlickScrollRect + точки-индикаторы.
  • Блок доверия — врезка в существующие auth-панели: код WhatsApp и телефон/код/2FA Telegram (QR-потока нет).
  • Успех — расширение WhatsappAuthSuccessPanel / TelegramAuthSuccessPanel; CTA ведёт в BotSettings → «Прайс-листы».
  • Чек-лист — префаб-карточка на BotsPage над списком ботов.

Состояние и анимация

  • Флаг PlayerPrefs "OnboardingSeen" — вне неймспейса ботов, не трогается при удалении бота; сбрасывается «Удалить все данные».
  • Шаги чек-листа выводятся из фактов: бот существует · канал авторизован · UploadedFilesStore непуст · первый входящий диалог с ответом бота; подпись канала — из isOnWhatsapp/isOnTelegram.
  • Переходы слайдов 0.3s OutCubic; чек успеха DOScale 0.9→1 OutBack; появление чек-листа — каскад 0.05s stagger.
  • Все размеры — в reference units канваса 1080×1920 (dp × 3), шкала типографики из unity-ui-builder.

Мокапы схематичны (1:4): финальные отступы и размеры — по шкале проекта при сборке билдером; иконки каналов условны.
