Описание Telegram API для сервиса Wappi
Методы отправки сообщений
Вы можете отправить сообщение в Telegram синхронной или асинхронной командой. Асинхронные методы выполняют запросы с постановкой в очередь, в ответе придет сообщение о постановке в очередь и id задачи. Синхронные методы отправления сообщений выполняются сразу и позволяют получить в ответе результат запроса
Особенности API
Это так называемый Telegram User API, который подойдет, если вы хотите использовать личный телеграм аккаунт в качестве userbot или для иных действий.
Токен авторизации передается в заголовке Authorization HTTP запроса. Profile_id передается query параметром в каждом запросе.
Безопасность
Мы используем API Telegram, который является частью экосистемы Telegram. Если вы используете наш API для флуда, спама, фейковых подписчиков, счетчиков просмотров каналов, Telegram может вас забанить. Рекомендуем использовать свои собственные socks5 прокси для подключения. 
Тестирование API
Чтобы протестировать API необходимо авторизоваться. Для этого на данной странице нажмите Authorize, в появившемся окне в поле Value вставьте ваш токен api. Токен можно найти в дашборде на странице вашего профиля

К изображению, видео или документу можно добавить подпись через параметр caption. Для документов можно указать отображаемое имя файла через параметр file_name.

Для пересылки необходимо указать идентификатор сообщения в параметре message_id и получателя в параметре recipient. При необходимости можно скрыть информацию об авторе исходного сообщения с помощью параметра drop_author.
После получения запроса сообщение помещается в очередь на отправку и обрабатывается системой в фоновом режиме. В ответе возвращается идентификатор задачи (task_id), который можно использовать для отслеживания статуса выполнения.
Request Body
 ПараметрТип данныхОбяз.Описание и пример
 message_id
string
Да
Идентификатор сообщения, которое необходимо переслать. Пример: "1059948"
recipient
string
Да
Получатель сообщения_\*_. В качестве значения можно указать номер телефона, имя пользователя, идентификатор пользователя или идентификатор чата. Для номера телефона допускаются различные форматы записи (со знаком +, пробелами, скобками, дефисами). Для имени пользователя допускается указание как с символом @, так и без него. Примеры: "79115556677", "@testname", "testname", "123456789"
drop_author
boolean
Нет
Определяет, будет ли скрыта информация об авторе пересылаемого сообщения. true — сообщение будет отправлено без указания автора и будет выглядеть как обычное сообщение, а не как пересланное, false — сохранить информацию об авторе. Пример: false

object
Нет
Объект с информацией о менеджере, от имени которого выполняется пересылка сообщения.

string
Нет
Идентификатор менеджера. Пример: "38200203"
manager.name
string
Нет
Имя менеджера. Пример: "Светлана"
manager.ava_link
string
Нет
Ссылка на аватар менеджера. Пример: "https://img.freepik.com/free-photo/view-adorable-persian-domestic-cat_23-2151773882.jpg"
manager.description
string
Нет
Дополнительная информация о менеджере. Пример: "описание пример"
 \* Отправка сообщений по идентификатору пользователя доступна только после того, как пользователь был найден по имени (username) или если с ним уже существует диалог.

 Частые вопросы 
Можно ли получить бан за использование сервиса? 
Мы используем API Telegram, который является частью экосистемы Telegram, и бана за использование этого API получить нельзя. Но мы не несем ответственности за неправомерное использование - если вы используете его для флуда, спама, фейковых подписчиков, счетчиков просмотров каналов, то Telegram может вас забанить. 
Можно ли производить массовые рассылки? 
Технически вы можете использовать Wappi для массовой рассылки, у нас нет ограничений на количество отправленных сообщений. Но мы не рекомендуем делать рассылку нежелательных сообщений или спама, иначе есть риск быть заблокированным алгоритмами антиспам системы Telegram.
