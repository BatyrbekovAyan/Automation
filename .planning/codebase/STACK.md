# Technology Stack

**Analysis Date:** 2026-06-23

## Languages

**Primary:**
- C# (Unity 6 scripting) — all game/application logic
- YAML (Unity scene serialization) — `Assets/Scenes/Main.unity` single-scene architecture

**Secondary:**
- Objective-C / Swift (iOS native bridges) — `Assets/Plugins/iOS/` for platform capabilities
- Java / Kotlin (Android native bridges) — `Assets/Plugins/Android/` for platform JNI calls

## Runtime

**Environment:**
- Unity 6 LTS (`6000.3.9f1`) — primary development/deployment engine
- .NET Framework 4.x (via Unity's Mono/IL2CPP backend)

**Build Targets:**
- iOS (primary target; uses IL2CPP AOT compilation)
- Android (secondary; supports both IL2CPP and Mono)

**Package Manager:**
- Unity Package Manager (UPM) — manages official Unity plugins, render pipeline, test framework
- NuGetForUnity (v3.9+) — manages C# NuGet packages in `Packages/nuget-packages/`
  - Config: `Packages/nuget-packages/NuGet.config`
  - Packages: `Packages/nuget-packages/packages.config`
  - Install location: InPackagesFolder (not Assets/)
  - Lockfile: `Packages/nuget-packages/packages.lock.json`

## Frameworks

**Core Rendering:**
- Universal Render Pipeline (URP) 17.3.0 — mobile-optimized forward rendering
- Canvas / UI Toolkit (uGUI 2.0.0) — all UI implemented via `Canvas` components and `RectTransform`

**Networking:**
- UnityWebRequest (built-in) — coroutine-based HTTP client for all REST API calls
- Newtonsoft.Json (NuGet, v13.0.4) — JSON serialization/deserialization (`JsonConvert.DeserializeObject<T>`)
- JsonUtility (built-in) — lightweight JSON parsing for `Secrets` data loading

**Animation & Tweening:**
- DOTween (Asset Store plugin, v2.2.8+) — UI animations and transitions
  - Location: `Assets/Plugins/Demigiant/DOTween/`
  - Used for: page slides, button presses, fades, cascading list effects

**Testing:**
- Unity Test Framework (UTF) 1.6.0 — EditMode and PlayMode test runner
- NUnit3 — test assertions and test runner format
- Test output: `Tools/test-output/` (headless) or `Temp/claude/test-summary.json` (Editor bridge)

**UI Components:**
- TextMeshPro (TMPro, built-in) — all text rendering (NO legacy Text component)
- RoundedCorners (UPM, ceorkm) — radius-based card styling without expensive `Mask`
- NativeShare (UPM, yasirkula) — native share sheets (iOS/Android)
- NativeCamera / NativeGallery (plugins, yasirkula) — media picker integration

**Build/Compilation:**
- Bee (built-in to Unity 6) — incremental C# compiler via Roslyn
- IL2CPP — ahead-of-time compilation for iOS; optional for Android
- Gradle — Android build system (generated by Unity)

**Editor Tooling:**
- Rider / Visual Studio 2022 (IDE plugins) — C# development and debugging
- mcp-unity (1.3.0, CoderGamester) — Claude integration for menu invocation and log parsing

## Key Dependencies

**Critical:**
- Newtonsoft.Json (13.0.4) — required for API response parsing; Unity's `JsonUtility` cannot deserialize complex nested objects
- UnityWebRequest — synchronous request/response pattern via `IEnumerator` + `yield return`
- URP 17.3.0 — mandatory for mobile performance; deferred rendering disabled

**Infrastructure:**
- DocumentFormat.OpenXml (3.4.1) — XLSX/DOCX file parsing for document uploads (business document support)
- ExcelDataReader (3.8.0) — read Excel files for bulk data import
- System.IO.Packaging (8.0.1) — archive/package handling for OpenXml
- System.Text.Encoding.CodePages (10.0.1) — legacy charset support for document text extraction

**Mobile Bridges:**
- NativeFilePicker (Yasirkula) — file selection dialog (iOS UIDocumentPickerViewController, Android Intent)
- NativeGallery (Yasirkula) — image/video gallery (iOS PHPickerViewController, Android ACTION_OPEN_DOCUMENT)
- NativeCamera (Yasirkula) — camera capture (iOS UIImagePickerController, Android Camera Intent)
- Plugins/Android/*.aar — pre-compiled Android libraries (not visible as dependencies, used via `AndroidJavaClass`)
- Plugins/iOS/*.a / *.h — pre-compiled iOS static libraries (linked at app build time)

**Plugins:**
- unity.webp (Asset Store) — WebP codec for sticker thumbnails (image compression)
- DOTween — tween animations (easing, timing)

## Configuration

**Environment:**
- Secrets loaded from `Assets/StreamingAssets/secrets.json` via `Secrets.cs`
  - Required keys: `wappiAuthToken`, `n8nAPIKey`, `telegramBotToken`, `greenApi.apiUrl`, `greenApi.idInstance`, `greenApi.apiTokenInstance`, `greenApiAvatar.*`
  - Template: `Assets/StreamingAssets/secrets.json.example` (checked in; `.gitignore` covers `secrets.json`)

**Build Configuration:**
- `ProjectSettings/ProjectSettings.asset` — build settings (target iOS 14.0+, Android API 29+)
- `ProjectSettings/EditorBuildSettings.asset` — scene list and build scenes (single scene: `Main.unity`)
- `ProjectSettings/GraphicsSettings.asset` — URP asset reference, quality settings
- `.gitattributes` — Git LFS tracking for binary assets (textures, audio, fonts, plugins)

**Package Resolution:**
- `.csproj` / `.sln` — auto-generated by Unity; `.gitignore`d
- Assembly definitions (`.asmdef`) — only in `Packages/` (UPM/NuGet assemblies); `Assets/` uses predefined `Assembly-CSharp`

## Platform Requirements

**Development:**
- **macOS** 12+ (Arm64 or Intel) — Unity Editor 6
- **Xcode** 14+ — iOS builds (Swift/Objective-C linking)
- **Android SDK / NDK** (via Android Studio or command-line tools)
- **Git LFS** — for binary asset storage

**Runtime (iOS):**
- iOS 14.0+ — minimum deployment target
- Required frameworks: `Foundation`, `UIKit`, `CoreServices`, `AVFoundation` (for media)
- Required capabilities: microphone, camera, photo library (controlled by `Info.plist`)

**Runtime (Android):**
- Android API 29+ (target API 34+)
- Required permissions: `INTERNET`, `READ_PHONE_STATE`, `ACCESS_FINE_LOCATION` (in `AndroidManifest.xml`)
- GMS (Google Mobile Services) optional for push notifications (not currently integrated)

**Network:**
- TLS 1.2+ for all HTTPS calls
- UnityWebRequest timeout: 30 seconds (hardcoded in networking layer)
