<objective>
Coalesce rapid incoming fragments into ONE «Вместе» suggestions request (BATCH-03). Add a pure, injectable-clock `IncomingDebounceGate` (mirrors `OpenChatLivePollGate`), drive it from `SuggestionsController.HandleLive` (reset the ~2.5s window on each incoming instead of firing immediately), and fire `IssueRequest` once when the window settles via a thin self-gating coroutine. `HandleManualRefresh` and `HandleCardTapped` stay IMMEDIATE; the pending window is cancelled (and `_pendingIncomingText` cleared) at ALL FOUR lifecycle boundaries — chat close (OnDisable), bot switch (ResetForNoOpenChat), same-bot chat switch (RestoreForActiveChat), and toggle-off (HandleToggle OFF branch). Everything is EditMode-verifiable — no live backend, no network call added (combine is free; the payload already ships the last <=12 messages).

Purpose: Without this, three fragments fire three back-to-back LLM suggestion requests; the seq guard hides the flicker but the cost + churn remain. This makes suggestions coalesce the same way the auto path does. The four-site cancel is load-bearing: a same-bot chat switch (chat A -> chat B) inside the window would otherwise let a pending fire carry chat A's `_pendingIncomingText` into a request scoped to chat B — a mixed-context call the `SuggestionSequenceGuard` cannot catch (the chatId is captured correctly at fire time; only the text is stale).
Output: `IncomingDebounceGate.cs` (new), `IncomingDebounceGateTests.cs` (new), `SuggestionsController.cs` (HandleLive + coroutine + 4 Cancel sites).

```csharp
/// <summary>Pure debounce decision for coalescing rapid incoming batches into one «Вместе»
/// suggestions request (BATCH-03). Reset on each incoming (Poke); fires ONCE when the window
/// settles (ShouldFire); Cancel drops a pending window on chat close / bot switch / same-bot
/// chat switch / toggle-off. The clock is injected (seconds) so "3 rapid pokes -> 1 fire after
/// the window" is EditMode-testable — mirrors the pure-seam discipline of OpenChatLivePollGate
/// (stateless), but this gate is STATEFUL (holds the deadline + armed flag across calls).</summary>
public sealed class IncomingDebounceGate
{
    /// <summary>The coalesce window, in seconds. ~2.5s per CONTEXT (single tunable; tune at e2e).
    /// Long enough to catch a burst of fragments typed ~1s apart, short enough to feel live.</summary>
    public const float WindowSeconds = 2.5f;

    private float _deadline;
    private bool _armed;

    /// <summary>Reset the window: fire WindowSeconds after the most recent incoming.</summary>
    public void Poke(float now) { _deadline = now + WindowSeconds; _armed = true; }

    /// <summary>Drop a pending window (chat close / bot switch) — never fires until re-Poked.</summary>
    public void Cancel() => _armed = false;

    /// <summary>True EXACTLY once when the window has settled; disarms so it fires only once.</summary>
    public bool ShouldFire(float now)
    {
        if (!_armed || now < _deadline) return false;
        _armed = false;
        return true;
    }
}
```
