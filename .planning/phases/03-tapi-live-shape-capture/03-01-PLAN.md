## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local shell → wappi.pro/tapi | The Wappi auth token crosses from the owner's machine to wappi.pro over HTTPS in the `Authorization` header. |
| secrets.json → capture script | The token is read from the gitignored secrets.json into a shell var; it must not cross into any output file, argv, or log. |
| capture script → git working tree | Response samples (may contain phone numbers / names / message bodies) are written to disk; the git boundary is controlled by `.gitignore`. |

Enforced by Task 1 acceptance grep `! grep -Eq '(echo|printf).*(TOKEN|wappiAuthToken)'`. |
| T-03-02 | Tampering | capture-shapes.sh endpoint set | mitigate | Read-only 8-endpoint allowlist; script contains none of message/send\|reply\|reaction, profile/add\|delete\|logout, sync/auth/*, webhook/*, and no `mark_all=` query param (which would mark chats read via a GET-shaped endpoint on the owner's real account). Enforced by Task 1 acceptance grep `! grep -Eq '/message/(send\|reply\|reaction)\|/profile/(add\|delete\|logout)\|/sync/auth/\|/webhook/\|mark_all='`. |
| T-03-03 | Information Disclosure | Tools/tapi/samples/ committed to git | mitigate | `Tools/tapi/samples/` added to `.gitignore`; committed SHAPES.md carries only structural verdicts + fully redacted mini-excerpts. Enforced by Task 2 acceptance grep `grep -q '^Tools/tapi/samples/$' .gitignore`. |
