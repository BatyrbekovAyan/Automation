<objective>
Close UAT defects **D1 (medium, ONB-04/05)** and **D3 (medium, ONB-04)** on the «Первые
шаги» checklist card — two facets of the same "the card is not driven by live facts" bug:

- **D1:** with ZERO bots, the card renders OVERLAPPING the EmptyState (repro: finish the
  carousel → wizard opens → press back). Fix: the card is hidden whenever no bots exist
  (EmptyState is the step-1 guidance); it first appears once ≥1 bot exists.
- **D3:** the checklist derives only on `OnEnable`, so right after bot creation the card shows
  all-unchecked until the user navigates away and back (the AddBotPanel is an OVERLAY on the
  still-active Bots page, so `OnEnable` never re-fires). Fix: an explicit `RefreshFromFacts()`
  invoked on every fact-changing moment, plus the existing `OnEnable`.

Both are solved by making `FirstStepsCard` externally refreshable (a static `Instance` + a
public `RefreshFromFacts()`), hiding via a CanvasGroup (so the root stays ACTIVE and reachable
even when visually hidden — a self-`SetActive(false)` root could never be re-shown), and
gating visibility on a new pure `FirstStepsCardVisibility.ShouldShow`. This plan is CODE-ONLY:
no scene mutation (the card root is already active in the scene; the CanvasGroup and the
BotsPage→card link are runtime/static, needing no new serialized fields).

Purpose: the checklist must be a live mirror of onboarding progress, and must never fight the
EmptyState.
Output: pure visibility rule + tests; a live-refreshable card; five fact-change hooks.
</objective>
