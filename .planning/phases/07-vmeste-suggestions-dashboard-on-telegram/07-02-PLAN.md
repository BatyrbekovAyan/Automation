## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-01 | Tampering / Injection | deep-link `OpenChat` resolving a server-returned `profileId` | mitigate | Resolution goes ONLY through the locally-built `DashboardProfileMap` (bots owned by the device). An unknown/forged `profileId` ⇒ `TryResolve` false ⇒ early-return (no navigation, no NRE). `channel` is taken from WHICH LOCAL entry matched — never from the server payload — so a server value cannot force a channel/bot switch to anything not already on-device. |
| T-07-02-02 | Tampering | server `chatId` passed to `SelectChat` | accept | `SelectChat` already soft-fails to the bot's list when the chat is absent (existing comment/behavior); `chatId` is a chat identifier, not a filesystem path — no traversal surface. Unchanged from shipped behavior. |
| T-07-02-03 | Information Disclosure | Telegram profile ids added to the POSTed `profileIds` | accept | The owner's own profile ids (globally-unique GUIDs, sentinel-stripped client-side; server Prep also strips "-1"); already-authenticated owner endpoint over HTTPS; same data class as the WhatsApp ids already sent. |
