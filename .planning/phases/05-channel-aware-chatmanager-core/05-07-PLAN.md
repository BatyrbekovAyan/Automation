<objective>
Close the three Telegram-only media PRESENTATION gaps found in device UAT (`05-HUMAN-UAT.md`). Classification is already coarse-but-correct from 05-06 — this plan adds the missing rendering treatments so each media type communicates the right message KIND, exactly as native Telegram shows it.

The three gaps (all Telegram-only; WhatsApp must stay byte-identical):
1. **.tgs animated sticker** — today renders as a document card (`AnimatedSticker.tgs 22 KB · TGS`). `.tgs` is gzipped Lottie, undecodable in Unity, so v1.1 target = a deliberate sticker-bubble PLACEHOLDER (borderless square, sticker glyph + «Стикер»), never a document card.
2. **Video note (кружок)** — today renders as a regular video card with a play overlay. Target = circular crop, no filename/card chrome, duration badge, existing tap-to-play.
3. **GIF** — today renders as a plain video. Target = keep the video pipeline (thumb + tap-to-play) + a "GIF" corner badge, no filename chrome.

Purpose: Telegram media reads correctly at a glance — a sticker looks like a sticker, a кружок looks like a кружок, a GIF is labelled — matching CHAT-03 presentation fidelity.
Output: Two new detection signals through the 4-layer pipeline (`isVideoNote`, `isGif`) + a mimetype rule (.tgs→Sticker), all channel-gated at Normalize; three additive `MessageItemView` treatments; full-matrix unit tests + WhatsApp regression net green.
</objective>
