<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| wappi tapi server → parser | Untrusted chats/filter + messages JSON (empty ids, wrong-typed timestamps, unknown types) is parsed and sliced |
| ActiveChannel → URL base | The channel selects api vs tapi; a wrong base sends a channel's profile_id to the other API |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-0503-01 | Denial of Service | ParseChatsJson display-name fallback | mitigate | `ChatIdFormat.DisplayFallback` never slices numeric/short/empty ids — retires the `chat.id[..^5]` ArgumentOutOfRange crash on TG ids (SHAPES.md Q5). |
| T-0503-02 | Tampering | last_timestamp/last_time parse | mitigate | Both parsed via `DateTimeOffset.TryParse`; a wrong-typed or absent field yields 0, never an exception (JsonUtility null-tolerant). |
| T-0503-03 | Spoofing | WappiEndpoints channel base | mitigate | URLs built from ActiveChannel through the single builder; both-channel tests (05-01) prevent cross-API leakage of profile ids. |
| T-0503-04 | Elevation of Privilege | Telegram chat/delete | mitigate | ActiveChannelSupportsChatDelete gates DeleteChat to a no-op on Telegram — no destructive server call is even attempted where no endpoint exists. |
</threat_model>
