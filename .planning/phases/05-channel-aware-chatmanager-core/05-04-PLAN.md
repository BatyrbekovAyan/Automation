<objective>
Implement the send-path tapi divergences. On Telegram: quoted replies use the dedicated `message/reply` endpoint (no `quoted_message_id`); reactions require a `recipient` in the body; mark-read drops the `mark_all` query and posts `{message_id}`; media sends use the channel-aware `EndpointFor`. Outbox entries snapshot the channel at send time so cross-session retries rebuild the right base URL. WhatsApp keeps identical URLs and bodies.

Send/mark-read base URLs (ChatManager.cs:1933, 2023) and the reaction URL (ReactionSend.cs:66) and media EndpointFor (MediaSend.cs:260) are all routed through 05-01's WappiEndpoints / channel-aware factory here (they were deliberately left for this plan so the URL swap and the body/query branch land together).

Purpose: a Telegram-authed bot can send text, media, quoted replies, reactions, and mark chats read — at WhatsApp parity.
Output: send-path branches across 3 ChatManager partials + outbox channel snapshot/retry + tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → wappi tapi (outbound POST) | text/reply/reaction/mark-read/media bodies built from chatId + message ids; auth header carries the token |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-0504-01 | Tampering | reply/reaction/mark-read request bodies | mitigate | Bodies serialized via JsonConvert (no string concatenation of untrusted values); recipient normalized through ChatIdFormat.Recipient; message_id is a server-issued stanza id. |
| T-0504-02 | Spoofing | send/mark/reaction/media base URL | mitigate | Base built from the channel (ActiveChannel for fresh sends, entry.channel for retries) via WappiEndpoints — a Telegram profile_id never posts to an api base. |
| T-0504-03 | Information Disclosure | auth token | accept | Token stays in the Authorization header (never in the URL/query), matching the established networking rule; no token logged. |
</threat_model>
