fix is defeated by tapi's current-state-only `reactions[]`. When the owner rapidly changes their OWN

server-me is already the NEWER differing emoji, so `Merge`'s never-clear-on-differ branch eats it for the

equality that ignores `time`) reports the lists equal — so `cached.reactions = merged` never runs and the
grace is never consumed through ANY of the three call sites. **The review is explicit: CR-01a and CR-02
must ship together — either alone leaves the defect live.**

**The fix (per the review, verbatim specs below):**
- **CR-01a — displaced-emoji discrimination.** The displaced pre-tap value already exists as `priorEmoji`
  (ChatManager.ReactionSend.cs:40, captured today only for the failure revert). Ride it ON the optimistic
  `MessageReaction` entry as a new JsonUtility-serializable `displacedEmoji` field (entries persist to disk,
  so a ChatManager-side map would vanish on relaunch while `time` survives — a map is WRONG). Merge then
  suppresses a differing server-me ONLY when it equals `displacedEmoji` (the genuine stale echo); any THIRD
  value (neither optimistic nor displaced) adopts + consumes freshness. Same-emoji confirm-adopt stays; the
  removal tombstone is gated the same way (displaced = the removed emoji).
- **CR-02 — the Reconcile seam.** Extract `Reconcile(cached, server, now, out renderChanged)` (Merge feeds
  it) and make `RefreshCachedMessageReactions` ALWAYS adopt the reconciled list; `renderChanged` (which
  considers the freshness/identity mutations Merge makes, not just the multiset) gates the event/repaint.
- **WR-01 — pin only.** The failed-POST revert re-arms freshness with the tap time but carries null
  displaced ⇒ adopt-on-differ resolves it as a side effect; pin with a test.
- **D15 (IN-01) — deterministic probe.** Editor-only one-shot at the first WhatsApp `type:"reaction"` raw:
  enqueue `raw.stanzaId` into the existing quote-resolve drain so the `[D15-probe]` report fires without
  owner choreography. Reuses the already-authed `messages/id/get` seam — zero secrets handling.
