## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| internet/mobile app -> /webhook/SetReplyMode | untrusted JSON body (profileIds, chatId, suppressed) crosses here |
| n8n -> Supabase Postgres | the Upsert node writes reply_mode_flags |
| mobile binary (anon key) -> Supabase Data API | the shipped anon key must never read/write this table |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | Upsert Postgres node | mitigate | Parameterized `$1,$2,$3::boolean` via `options.queryReplacement` — never string-concat profileId/chatId into the SQL |
| T-09-02 | Tampering / DoS | /webhook/SetReplyMode (unauthenticated) | accept | R-02-01 accepted-risk posture, consistent with every app `/webhook/*` (UploadFile/DeleteFile/DashboardOutcomes). Blast radius = flip a bot's reply suppression; no data exfil, no LLM cost. Authentication is a deferred item |
| T-09-03 | Tampering | Validate Code + If invalid? | mitigate | Malformed body -> `{ invalid:true }` -> Respond-error BEFORE the Upsert node runs (no partial write); 09-04 curl matrix asserts it |
| T-09-04 | Information Disclosure | reply_mode_flags RLS | mitigate | Default-deny RLS (`enable row level security`) + `revoke all from anon, authenticated`; the mobile anon key can never reach the table (only the server Postgres cred does), mirroring conversation_outcomes |
</threat_model>
